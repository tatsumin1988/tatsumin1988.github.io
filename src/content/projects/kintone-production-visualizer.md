## 背景

生産実績は Kintone 上に蓄積されていたものの、入力済みデータを判断に使うための見え方が弱く、必要な切り口で都度確認するのが手間になっていました。ラインごとの差や、製品ごとの効率推移を見たい場面でも、単純な一覧だけでは流れをつかみにくい状態でした。

## 解決アプローチ

Kintone API を使って実績データを取得し、日単位・製品単位で再集計したうえで、Streamlit 上に可視化用の画面を構築しました。概要、製品別推移分析、ライン別分析、詳細データというタブ構成にし、見る目的に応じて切り替えやすい設計にしています。

## 工夫した点

### 500件制限を超える全件取得

Kintone API の標準制限に引っかかると途中までしか見えないため、ページネーションを組み込んで全件取得できるようにしました。

### 日別集計で比較しやすくした

同じ製品・同じ日のデータをまとめて 1 日 1 点にすることで、トレンドを読み取りやすい形に整理しています。

### フィルタで現場の見方に寄せた

「全ライン」と個別ラインの切り替え、効率 100% 未満だけの抽出など、現場で見たい条件から逆算して UI を組みました。

## 効果

- どの製品やラインで効率が落ちているかを追いやすくなった
- 生データをそのまま読むより、日次の傾向を把握しやすくなった
- Kintoneの蓄積データを、判断に使える可視化へ変換できた

## 今後の改善

今後は、目標値との比較、定期レポート出力、通知連携などを追加し、見るだけでなく改善アクションへつながる仕組みに拡張できます。
